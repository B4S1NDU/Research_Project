# Curator Verification System - End-to-End Test Guide

This guide walks through testing the complete curator verification workflow where generated AI content receives curator approval and badge verification is displayed to end-users.

## System Overview

**Architecture:**
- Admin Panel (port 5002) - Curator interface for reviewing and approving AI-generated content
- Basi-Component2 (port 5000) - Artifact explanations with curator verification badges
- Basiii (port 5001) - Artifact scenarios with curator verification badges
- Main Dashboard (port) - Entry point for all components

**Workflow:**
1. User requests AI-generated content (explanation or scenario)
2. AI generates content and saves to moderation queue
3. Curator reviews in admin panel
4. Curator approves and publishes
5. End-users see curator verification badge on subsequent requests

## Prerequisites

- All three services running (admin_server 5002, Basi-Component2 5000, Basiii 5001)
- Default curator account: `curator` / `curator123`
- Default admin account: `admin` / `admin123`

## TEST 1: Explanation Verification (Basi-Component2)

### Step 1: Generate an Explanation (Create Queue Item)

1. Open http://localhost:5000
2. Click any artifact to view details
3. Scroll down to "AI Analysis" section
4. Click "Generate AI Explanation" button
5. Wait for AI analysis to complete
6. **EXPECTED:** Explanation shows but NO curator badge (not yet approved)

### Step 2: Review in Curator Panel

1. Open new tab: http://localhost:5002/panel/
2. Login with `curator` / `curator123`
3. Click "Explanations" tab
4. Find the explanation you just generated
   - Should show artifact name, generated timestamp, "Pending Review" status
5. Click the explanation row to expand or select it

### Step 3: Approve in Curator Panel

1. In the Explanations queue, find your generated explanation
2. Click "Approve" button
3. Enter curator notes (e.g., "Accurate and well-sourced") - optional
4. Click "Confirm" button
5. **EXPECTED:** Status changes to "Approved", green checkmark appears

### Step 4: Check Verification Display in End-User Interface

1. Go back to http://localhost:5000
2. Stay on the same artifact
3. Scroll to "AI Analysis" section
4. Click "Regenerate" to request the explanation again
5. **EXPECTED RESULT:** 
   - Explanation text appears
   - Green verification badge shows: "✓ Verified by Curator"
   - Verified by: "curator" (or the curator's actual name)
   - Footer changes from "Generated by AI" to "Curator-verified AI analysis"

## TEST 2: Scenario Verification (Basiii)

### Step 1: Generate a Scenario (Create Queue Item)

1. Open http://localhost:5001
2. Click any artifact
3. Select a scenario from the list (e.g., "Museum Exhibition", "Restoration Workshop")
4. Click the scenario card
5. Wait for analysis to generate in ScenariosPage
6. **EXPECTED:** 3-topic analysis shows but NO curator badge initially

### Step 2: Review in Curator Panel

1. Keep the Basiii page open in one tab
2. Open http://localhost:5002/panel/ in another tab
3. Login if needed with `curator` / `curator123`
4. Click "Scenarios" tab
5. Find the scenario you just generated
   - Display shows: artifact name, 3 analysis topics, model version, "Pending Review" status

### Step 3: Approve in Curator Panel

1. In Scenarios queue, find your generated scenario
2. Click "Approve" button
3. Optional: Enter curator notes
4. Click "Confirm"
5. **EXPECTED:** Status changes to "Approved", approval timestamp recorded

### Step 4: Check Verification Display in End-User Interface

1. Go back to Basiii tab with your scenario
2. Click "Regenerate Analysis" button
3. Wait for new analysis to generate
4. **EXPECTED RESULT:**
   - Green curator badge appears at top: "✓ Verified by Curator"
   - Verified by: "curator"
   - All 3 analysis topics display with curator-approved content

## TEST 3: Verify Audit Trail

1. In Admin Panel, click "Audit Log" tab
2. Scroll to see recent entries
3. **EXPECTED:**
   - "SCENARIO_SAVED" entry when you generated scenario
   - "SCENARIO_APPROVED" entry when you approved
   - "EXPLANATION_SAVED" entry when you generated explanation
   - "EXPLANATION_APPROVED" entry when you approved
4. Each entry shows timestamp, curator name, action, entity details

## TEST 4: Curator-Edited Content Display

This advanced test verifies that curator-edited content is served to users:

1. **Generate scenario or explanation** (see tests 1-2)
2. **Approve in curator panel** but also make edits:
   - Scenario: Edit one or more of the 6 fields (3 topics + 3 descriptions)
   - Explanation: Edit the explanation text
3. **Save/Publish** the edited version
4. **In end-user interface:** Regenerate/request again
5. **EXPECTED:** Your curator edits are displayed to the user, not the original AI-generated text

## TEST 5: Rejection Workflow

1. **Generate new content** (explanation or scenario)
2. In Admin Panel, click "Reject" instead of "Approve"
3. Add rejection reason (e.g., "Contains inaccuracy about material composition")
4. In end-user interface, regenerate
5. **EXPECTED:** 
   - No curator badge (rejected content not served)
   - Generates new AI content for review
   - Original rejection remains in audit log

## TEST 6: Admin Publishing

1. In Admin Panel, approve content as curator
2. Scroll to see "Publish" button (admin-only)
3. **As curator:** Button should be grayed out or not visible
4. Logout, login as `admin` / `admin123`
5. Navigate to approved content
6. Click "Publish" button
7. **EXPECTED:** Status changes to "Published", available for all users

## TEST 7: Multi-Curator Verification

1. **Approve as curator:**
   - Login to http://localhost:5002 as `curator` / `curator123`
   - Generate and approve content
   - Verify name shows as "curator" in badge

2. **Edit as different curator** (if available):
   - Add additional curator accounts in Admin Panel

   - Verify each curator's name appears correctly in audit log

## Troubleshooting

### Badge Not Appearing
- **Check:** Is moderation enabled? Both backends should have `_MODERATION_ENABLED = True`
- **Check:** Did you click Approve (not just reject)?
- **Check:** browser cache - try hard refresh (Ctrl+Shift+R)
- **Check:** Admin Panel shows "Approved" status

### API Errors
- **Check:** All 3 ports running (5000, 5001, 5002)
- **Check:** admin_db.py exists in project root
- **Check:** Flask server console for Python errors

### Content Not Updated
- **Issue:** User cache showing old AI-generated version
- **Solution:** Hard refresh browser or clear cache
- **Alternative:** Sign out and back in

### Curator Panel Shows Empty Queues
- **Check:** Verify CORS is enabled in admin_server.py
- **Check:** Did you generate content first? (must generate to populate queue)
- **Check:** Admin server logs for errors

## Response Format Examples

### Explanation Response (from Basi-Component2 /api/artifacts/<id>/explain)

**Before Curator Approval:**
```json
{
  "explanation": "Bronze artifacts were crafted using...",
  "curator_verified": false,
  "verified_by": null,
  "curator_notes": null,
  "verification_date": null,
  "cached": false
}
```

**After Curator Approval:**
```json
{
  "explanation": "Bronze artifacts were crafted using... [possibly edited by curator]",
  "curator_verified": true,
  "verified_by": "curator",
  "curator_notes": "Accurate historical information with good sources",
  "verification_date": "2026-03-02T14:32:10.123456",
  "cached": false
}
```

### Scenario Response (from Basiii /api/generate)

**Before Curator Approval:**
```json
{
  "artid": "A001",
  "scenario_id": "sce123",
  "answerTopic1": "Exhibition Design",
  "answerDescription1": "Showcase the artifact...",
  "answerTopic2": "...",
  "curator_verified": false,
  "verified_by": null
}
```

**After Curator Approval:**
```json
{
  "artid": "A001",
  "scenario_id": "sce123",
  "answerTopic1": "Exhibition Design",
  "answerDescription1": "Showcase the artifact in modern context...",
  "answerTopic2": "...",
  "curator_verified": true,
  "verified_by": "curator",
  "curator_notes": "Well-researched exhibition approach"
}
```
